Paituli STAC - Spatio Temporal Asset Catalog

STAC is a specification to describe geospatial datasets with temporal dimension. It drastically simplifies searching and downloading datasets. STAC includes only the metadata of datasets and links to the actual files. The data files themselves are usually stored in the cloud. STAC is most often used for remote sensing imagery and other raster data, but it can be used also for vector and LiDAR data. STAC is especially suitable for time-series applications.

STAC main concepts:

  • Catalog - general description of the service and links to available Collections. Example: Paituli STAC
  • Collection - collection specific general metadata and links to available Items. In one collection are similar items. Example: ESA/SYKE, Sentinel-2 monthly index mosaics
  • Item - the core atomic unit, describing data for specific time and location. Each Item has one or several Assets. Example: ESA/SYKE, Sentinel-2 monthly index mosaics of area X at timepoint Y
  • Asset - Files and links related to an Item, can be data files or links to metadata etc. Example: NDVI index file of ESA/SYKE, Sentinel-2 monthly index mosaics of area X at timepoint Y. The dataset includes several indices, each index is an Asset, because each index is in a separate file.

Globally, a lot of STAC catalogs exist, STAC index website provides a list of public STAC catalogs. STAC index provides also a good possibility to explore different catalogs.

When searching STAC catalogs, the main criteria are location, time and collection name. Items may have also additional information, for example cloud coverage for optical remote sensing images, that also can be included to search criteria.

STAC can be provided as static collection of JSON files or as an API. STAC API support fast searching of big datasets with thousands or millions of Items.

The best storage option for STAC data is a cloud optimized format: Cloud-Optimized GeoTiff (COG) for raster and COPC for LiDAR. Cloud optimized formats enable downloading only a subset of larger files, covering only a defined area or only generalized overview of the data. Also other formats can be used, but depending on file inner organization subsetting might work less optimally.

For searching and downloading data from STAC some external tool is required. STAC end-user tools are available for Python , R, QGIS and Julia. Python and R have further packages for downloading and analyzing multitemporal raster data. These tools also support parallel computing, so handling bigger datasets is easily possible. STAC browser can be used for searching data from STAC with web-browser. Each STAC catalog has its own STAC end-point, which should be given to the tools, so that they can find the catalog. For more details see the examples mentioned at the end of this page.

Paituli STAC contents

Paituli STAC catalog includes Finnish data, currently:

  • Stored at CSC:
    • ESA, Sentinel-2 products, processed to Level-2A (Surface Reflectance), a selection of mostly cloud-free products from Finland. Downloaded to CSC Allas by Maria Yli-Heikkilä (LUKE), Arttu Kivimäki (NLS/FGI) and Matias Heino (Aalto). Files in JPG2000 format.
  • Stored at FMI, from FMI Tuulituhohaukka static STAC catalog, all files COGs.:
    • ESA/FMI, Sentinel-2 11-days surface reflectance mosaics.
    • ESA/FMI, Sentinel-1 daily backscatter mosaics: VV and VH polarisation.
    • ESA/FMI, Sentinel-1 11 days backscatter mosaics: VV and VH polarisation.
    • ESA/FMI, Sentinel-2 annual surface reflectance mosaics.
    • ESA/SYKE, Sentinel-2 monthly index mosaics: NDVI, NDBI, NDMI, NDSI, NDTI.
    • ESA, Sentinel-1 backscatter tiles: VV and VH polarisation.
    • USGS/SYKE, Landsat (4 and 5) yearly index mosaics: NDVI, NDBI, NDMI, NDSI, NDTI. This collection unfortunately has techinical problems on FMI side, so data download likely does not work.
    • NLS, Digital terrain model products: DTM, aspect, slope.
    • Finnish Forest center, Canopy height model.
    • LUKE, Multi-source forest inventory products.
    • LUKE, Forest wind damage risk map.
    • FMI, Daily wind damage risk map.

Paituli is a STAC API and its end-point is: https://paituli.csc.fi/geoserver/ogc/stac

STAC browser for Paituli STAC, see from web-browser what data collections are available, search for items and also preview COG files.

CSC has prepared example scripts for Python and R, that demostrate how to search and use STAC data and what are the main required libraries.

Paituli STAC is in early phases of operation, so more data collections will be added. Next, Geoportti geocubes and Paituli own data will be added.

Paituli STAC has been developed using Academy of Finland Geoportti funding.
