Bulk download of files from Paituli

Paituli Download page enables downloading data as .zip file max. 3 Gb at a time. This can be limiting, if the needed dataset is bigger. This page describes how to download bigger datasets as files. Alternative options are to use OGC APIs or STAC, to get access to the data.

For bigger downloads, there are two main options:

  • Download a list of selected files. The file list can be created on Download data page.
  • Download a folder, possibly including subfolders.

The files are available via three different protocols:

  • HTTPS: https://www.nic.funet.fi/index/geodata/
  • FTP: ftp://ftp.funet.fi/index/geodata/
  • rsync: rsync://rsync.nic.funet.fi/ftp/pub/sci/geo/geodata/

FTP and rsync are in some organizations limited by firewalls. HTTPS should always work.

Choose the right option for you, depending on your operating system, firewall limitations and download interests:

Download type Windows Linux, Mac, Windows WSL
List of files Paituli PowerShell script (HTTPS) rsync, wget (HTTPS)
Folder WinSCP, FileZilla (both FTP) FileZilla (FTP), rsync, wget (FTP or HTTPS)

If you want to download a folder, but have Windows and can not use FTP/rsync, then there is not suitable tools to download a folder. As workaround, create a .txt file with the folder name in it and use the Paituli PowerShell script to download the folder.

Download a list of files

If you want to download only specific mapsheets of some dataset, you need to first create a list of files in Paituli and then download the files with a commandline tool.

Creating the list of files

  1. Open the Download data page.
  2. Select the dataset you are interested
  3. Select the mapsheets you need from the map or find them with the search.
  4. Click on the "Download list of files" button on the left side.
  5. You will receive the link to file list to your e-mail, download that file.

Alternatively you can make a custom file using the paths given in index map, which is available for each dataset in Download data page Links tab.

Downloading the files with a list of files.

Unfortunately the graphical tools do not support downloading with a list of files, but is possible with command-line tools. For Windows users we provide custom PowerShell script. Linux and Mac users can download with wget or rsync , which should be always available by default.

Paituli PowerShell script for Windows

For Windows users we provide PowerShell script to download files via HTTPS. This works ONLY in Windows. The script creates new folders, to keep the original folder structure. The script skips already downloaded files.

Steps to use the script:

  1. Download the files list.
  2. Download the Paituli download script.
  3. Open this script in a text editor (for example NotePad) and change the paths marked with CHANGE. Save.
  4. Open PowerShell.
  5. Move to the folder where you saved the script, use cd command
  6. Run the script:
    .\download_paituli_data.ps1

wget

Use -i option to give the name of files list.

wget -i file_list.txt -P local_folder_to_save/

rsync

For rsync, remove from the beginning of each row 'http://www.nic.funet.fi/index/' in the file list.

rsync -a --files-from=file_list.txt rsync://rsync.nic.funet.fi/ftp/pub/sci/geo local_folder_to_save/

Download a folder

This is a relatively easy option, if the download needs match with the folder structure. Each Paituli dataset has its own folder, so this suits well for downloading a full dataset. Some bigger datasets have also several sub-folders and then it is case specific, if the sub-folders match your download needs.

Finding the path of the folder

For finding the folder you can simply browse the directories in HTTPS or SFTP mode. The datasets have been stored in logical structure, so you might find what you are looking for. In the tree you have to first select the data producer, then dataset and then year (and other options). The folders have mostly names in Finnish.

The other option is to find the dataset specific path from Download data page:

  1. Select the dataset you are interested
  2. Open the "Links" tab in lower left corner. The links for all 3 protocols are displayed.

In the folder of each dataset is a Readme-file, that includes the basic infomartion and link to Etsin, where full dataset descriptions are available.

Downloading the files of a folder and its possible subfolders

Different tools can be used for downloading a folder. The listed options here download automatically subfolders and keep the same structure on local computer.

  1. The easiest is a graphical FTP tool like FileZilla or WinSCP (only for Windows).
  2. rsync or wget provide more options and can continue interrupted download without downloading everything again. rsync and wget are included in most Linux and Mac distributions by default. In Windows you could use Windows Subsystem for Linux.

Graphical FTP tools

  1. Connect to server. host name: ftp.funet.fi, protocol: FTP, port number: 21.
  2. If username or password are asked, leave the fields empty.
  3. Navigate on the right side to the folder, where the dataset is stored.
  4. Open on the left side the folder, where you want to move the files.
  5. Drag the files or folders on the right side to left for download.

rsync

rsync -a rsync://rsync.nic.funet.fi/ftp/pub/sci/geo/geodata/mml/hallintorajat_milj_tk/2017/ local_folder_to_save/
  • Change the blue parts in the command as needed.
  • -a use archive mode, inc. save the original dates and download recursively also all subfolders

wget

In some places FTP and rsync are forbidden at firewall level, then you can use HTTPS with wget. wget has a lot of different options, one well working combination is this:

wget -r -l inf -N -np -nH -x -c --cut-dirs=6 ftp://ftp.funet.fi/pub/sci/geo/geodata/mml/hallintorajat_milj_tk/2017/ -P local_folder_to_save/
wget -r -l inf -N -np -nH -x -c --cut-dirs=4 https://www.nic.funet.fi/index/geodata/mml/hallintorajat_milj_tk/2017/ -P local_folder_to_save/
  • -r, recursive download
  • -l inf, how deep the recursive search goes, default is 5, here set to infinite
  • -N, update only, do not download already existing files, this is important if download was interrupted or updating already existing data.
  • -np, do not download parent directories
  • -nH, remove hostname
  • -x, make directories similarly to Paituli
  • -cuts-dirs, cut certain number of directories from the beginning to avoid too deep directory trees
  • -c, continue broken download
  • Use the ftp protocol if possible, otherwise you might get some extra index.* files.
